01
SQL Server Triggers Audit Log
PhilHealth Ledger Audit Trigger — psPHICLedgers

Wrote and deployed an AFTER UPDATE trigger on the PhilHealth main ledger table of a live Hospital Information System. Logs field-level changes per column into a centralized audit table — tracking who changed what and when, in Philippine time (UTC+8).

  • Tracks 5 columns: isdelete, caseRateHBAmount1/2, caseRatePFAmount1/2
  • Resolves modifier name via JOIN to appsysUsers and psPersonalData
  • NULL-safe comparisons using ISNULL() to prevent false-positive logs
  • Separate INSERT per column — only fires when that specific field changes
  • TriggerSource flag set to 'MAIN' for filtering in audit log queries

SQL Triggers PhilHealth
PhilHealth PF Ledger Audit Trigger — psPHICLedgersPF

Companion trigger to Project 01, written for the Professional Fee (PF) sub-ledger table. Logs changes to PhilHealth type, PF amount, and covered services — linking back to the main ledger via foreign key for full audit traceability across both tables.

  • Tracks 3 columns: phictype, PFamount, phicservices
  • Links PF table to parent psPHICLedgers via FK_TRXNO_psPHICLedgers
  • NULL-safe 3-condition WHERE clause for phictype and phicservices changes
  • TriggerSource set to 'PF' — separates PF logs from MAIN logs in audit queries
  • Stores PF_TRXNO alongside FK_TRXNO for granular row-level traceability

SQL SSRS Reporting
Patient PN Balance Report — SSRS Dataset Queries

Three parameterized SQL queries built as SSRS datasets for a Promissory Note balance report. Given a patient MRN parameter, the report pulls total PN amount, Official Receipt cash payments, and Credit Memo payments from separate tables into a unified finance-ready output.

  • Dataset 1: PN summary using a subquery to aggregate billtrancode credits (PNHB, PNPF, PNPR, PNADJ)
  • Dataset 2: Official Receipt payments via multi-table JOIN across faCRMstr, faArinv, emdPatients
  • Dataset 3: Credit Memo payments grouped by doctype and dmcmno with SUM aggregation
  • All datasets filtered by @MRN parameter for SSRS drill-down use
  • HAVING clause ensures only patients with outstanding PN balance appear

01_phic_ledger_audit_trigger.sql — sample 
-- Log caseRateHBAmount1 if changed
INSERT INTO dbo.philhealthLogs
    (FK_TRXNO, columnname, modifiedDate, modifiedBy, oldvalue, newvalue, TriggerSource, tablename)
SELECT
    i.PK_TRXNO,
    'caseRateHBAmount1',
    FORMAT(DATEADD(HOUR, 8, SYSUTCDATETIME()), 'MMM dd yyyy h:mmtt'),
    CONCAT(pd.lastname, ', ', pd.firstname, ' ', pd.middlename),
    ISNULL(d.caseRateHBAmount1, 0),
    ISNULL(i.caseRateHBAmount1, 0),
    'MAIN',
    'psPHICLedgers'
FROM inserted i
INNER JOIN deleted d       ON i.PK_TRXNO = d.PK_TRXNO
INNER JOIN appsysUsers au  ON i.editby = au.usercode
INNER JOIN psPersonaldata pd ON au.PK_appsysUsers = pd.PK_psPersonalData
WHERE ISNULL(i.caseRateHBAmount1, 0) <> ISNULL(d.caseRateHBAmount1, 0);
